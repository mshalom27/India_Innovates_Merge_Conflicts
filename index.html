Dynamic Routing Algorithm

Custom A* / Dijkstra's graph traversal with live AI-weighted edges:

w(u,v) = α·d(u,v) + β·ρ(u,v) + γ·s(u,v)
d(u,v)Physical distance between intersections u and v
ρ(u,v)Live vehicle density from YOLO camera
s(u,v)Signal phase penalty (current red/green state)
α,β,γTunable weights — β & γ heavily prioritized for emergencies

Judge Q&A

Edge Cases &
Fail-Safes

AURA-GRID anticipates real-world failure modes. Every "What if?" has an engineered answer.

🚑What if two ambulances approach from opposite directions?↓
The system assigns priority via a FIFO queue. If hospital integration is active, the vehicle carrying the more critical patient (e.g., Cardiac Arrest > Broken Arm) receives priority. The secondary ambulance gets a 3-second delayed corridor activation.
🌧️What if the camera goes down or is covered in rain?↓
The system automatically detects a null or degraded camera feed and reverts to the pre-programmed fixed-timer fallback mode. An alert is sent to the control center. Normal AI operation resumes automatically when the feed is restored.
🔒How do we prevent abuse of the VVIP system?↓
Every "Green Corridor" request is immutably logged with a timestamp, user ID, GPS track, and session token. MFA is required for VVIP-tier access. All logs are auditable and tamper-evident. Unauthorized activation triggers an immediate alert to senior administrators.
📡What if the internet connection is lost mid-corridor?↓
Each intersection node caches the corridor plan locally at activation. If cloud connectivity drops, the node executes its pre-cached green schedule autonomously until the last downloaded instruction expires, then defaults to safe fixed-timer mode.
🚶Does this worsen pedestrian safety at crossings?↓
No. The dynamic time algorithm enforces a non-negotiable minimum 15-second pedestrian phase regardless of vehicle density. Emergency overrides always trigger the 3-second yellow safety buffer, giving pedestrians at the crossing time to clear before the green switches to the emergency lane.
⚖️Is this scalable to an entire city?↓
Yes. The architecture uses edge AI at each node, so processing is distributed — no single server bottleneck. The central server only handles coordination signals (WebSocket messages), not video processing. Adding a new intersection node is a hardware + config operation, not a software re-architecture.
